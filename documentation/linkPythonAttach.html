Attach.py

The python Attach.py file provides with an interface to populate the Entities dictionary (see Definitions.py for more details). This file contains several functions to populate specific fields.

  • addNode(tag, ndof, coords, freedof, totaldof):

    • tag : The identifier of this node, i.e., tag > -1
    • ndof : Number of degree of freedom, i.e., ndof > 0
    • coords : Coordinates of the node, i.e., (x,y) or (x,y,z)
    • freedof : The free (with restrain/constraint) degree of freedom numbering of this node
    • totaldof : The total (without restrain/constraint) degree of freedom numbering of this node

    Example

    A NODE can be defined using the python interface as follows:
    SVL.addNode(tag=6, ndof=3, coords=[1.00, 0.00, 0.00])

  • addMass(tag, dof, vals):

    • tag : The identifier of the Node to add mass, i.e., tag > -1
    • dof : Degree of freedom number where the mass will be added, i.e., ndof > 0
    • vals : Mass values to be added to specific degree of freedom

    Example

    A MASS can be defined using the python interface as follows:
    SVL.addMass(tag=2, dof=[1, 2], vals=[1.00, 1.00])

  • addRestrain(tag, dof):

    • tag : The identifier of the node, i.e., tag > -1
    • dof : Number or list of degree of freedom to be restrained.

    Example

    A RESTRAIN can be defined using the python interface as follows:
    SVL.addRestrain(tag=6, dof=[1, 2, 3])

  • addConstraint(tag, name, attributes):

    • tag : The identifier of the constraint, i.e., tag < -1
    • name : The constraint type that is being applied.
    • attributes : Dictionary containing the master node(s) information. The fields are
      • 'stag' : The slave node(s) tags to be constrained
      • 'sdof' : The slave degree-of-freedom to be constrained
      • 'mtag' : The master node(s) tags to be employed
      • 'mdof' : The master degree-of-freedom to be combined
      • 'factor': The combinational factors to be combined

    Example

    A CONSTRAINT can be defined using the python interface as follows:
    SVL.addConstraint(tag=-2, name='Equal', attributes={'stag': 4, 'sdof': 1, 'mtag': 3, 'mdof': 1})

  • addDiaphragm(tag, attributes):

    • tag : The identifier of the diaphragm, i.e., tag > 1
    • attributes : Dictionary containing the diaphragm action and Nodes
      • 'tag' : The node tag that the diaphragm node will take (must be different from 'Nodes')
      • 'list' : List of Nodes that belong to the diaphragm
      • 'axis' : Axis that is perpendicular to the diaphragm

    Example

    A DIAPHRAGM can be defined using the python interface as follows:
    SVL.addDiaphragm(tag=1, attributes={'tag': 9, 'axis': 'Z', 'list': [5,6,7,8]})

  • addRigidBody(tag, attributes):

    • tag : The identifier of the diaphragm, i.e., tag > 1
    • attributes : Dictionary containing the diaphragm action and Nodes
      • 'tag' : The node tag that the rigid body node will take (must be different from 'Nodes')
      • 'ndof' : The number of degree of freedom of the rigid body i.e., 3 (2D) and 6 (3D)
      • 'list' : List of Nodes that belong to the rigid body
      • 'center' : Center of rotation of the rigid body

    Example

    A RIGID BODY can be defined using the python interface as follows:
    SVL.addRigidBody(tag=1, attributes={'tag': 9, 'ndof': 6, 'center': [0.1, 0.1, 0.1], 'list': [5,6,7,8]})

  • addSupportMotion(tag, attributes):

    • tag : The identifier of the restrained node, i.e., tag > 1 to apply the support motion
    • attributes : Dictionary containing the support motion action on a Nodes in a direction
      • 'type' : Constant or TimeSerie
      • 'value' : Displacement value if type=static
      • 'file' : Displacement time serie file if type=dynamic
      • 'dof' : Degree of freedom to apply the support motion

    Example

    A SUPPORT MOTION can be defined using the python interface as follows:
    SVL.addSupportMotion(tag=708, attributes={'file': 'Ricker.txt', 'type': 'TimeSerie', 'dof': 1})

  • addMaterial(tag, name, attributes):

    • tag : The identifier of this material, i.e., tag > -1
    • name : Seismo-VLAB material class name
    • attributes : Specific properties for the created material, for example
      • name=Viscous1DLinear:
        'eta' : is the linear viscous coefficient.
      • name=Elastic1DLinear, Elastic2DPlaneStress, Elastic2DPlaneStrain, Elastic3DLinear:
        'E' : is the elastic modulus of elasticity.
        'nu' : is the Poisson’s ratio.
        'rho' : is the material density. Can be omitted for static analysis.
      • name=PlasticPlaneStrainJ2, Plastic3DJ2:
        'K' : is the bulk’s modulus.
        'G' : is the elastic shear modulus.
        'rho' : is the material density.
        'H' : is the hardening modulus.
        'Sy' : is the yield stress.
        'beta' : is the Integration parameter or the Kinematic/Hardening ratio.
      • name=PlasticPlaneStrainBA, Plastic3DBA:
        'K' : is the bulk’s modulus.
        'G' : is the elastic shear modulus.
        'rho' : is the material density.
        'H0' : is the hardening modulus.
        'Su' : is the Undrained soil strength.
        'beta' : is the Integration parameter or the Kinematic/Hardening ratio.
        'h' : the Exponential hardening modulus parameter.
        'm' : is the Exponential hardening modulus parameter.

    Example

    A MATERIAL can be defined using the python interface as follows:
    SVL.addMaterial(tag=1, name='Elastic1DLinear', attributes={'E': 4.00, 'nu': 0.00, 'rho': 0.00000})

  • addSection(tag, name, model, attributes):

    • tag : The identifier of this section, i.e., tag > -1
    • name : Seismo-VLAB section class name
    • model : model of the section, i.e., model=plain,fiber
    • attributes : Specific properties for the created section, for example
      • model=Plain:
        material is the material identifier for the cross section.
        • name=Lin2DCircular,Lin3DCircular:
          'r' : is the radius of the line cross section.
        • name=Lin2DRectangular, Lin3DRectangular:
          'h' : is the hight of the line cross section.
          'b' : is the width of the line cross section.
        • name=Lin2DAngle,Lin2DChannel, Lin2DWideFlange, Lin2DTee, Lin2DRectangularTube, Lin3DAngle,Lin3DChannel, Lin3DWideFlange, Lin3DTee, Lin3DRectangularTube:
          'h' : is the hight of the line cross section.
          'b' : is the width of the line cross section.
          'tf' : is the flange thickness of the line cross section.
          'tw' : is the web thickness of the line cross section.
        • name=Lin2DCircularTube, Lin3DCircularTube:
          're' : is the thickness of the area section.
          'ri' : is the thickness of the area section.
        • name=Lin3DThinArea:
          'th' : is the thickness of the area section.
      • model=Fiber:
        this feature is under development.

    Example

    A SECTION can be defined using the python interface as follows:
    SVL.addSection(tag=2, name='Lin3DThinArea', model='Plain', attributes={'material': 2, 'th': 0.1})

  • addElement(tag, name, conn, attributes):

    • tag : The identifier of this element, i.e., tag > -1
    • name : Seismo-VLAB element class name
    • conn : Connectivity array of this element
    • attributes : Specific properties for the created element, for example
      • name=ZeroLength1D:
        • 'material' is material identifier.
        • 'dir' is the direction of action (X,Y,Z = 1, 2, 3).
      • name=UnxBoucWen2DLink, UnxBoucWen3DLink:
        • 'alpha' Post-yield ratio
        • 'eta' Yielding exponent (sharpness of hysteresis loop corners)
        • 'beta' First hysteretic shape parameter
        • 'gamma' Second hysteretic shape parameter
        • 'tol' Newton-Raphson Tolerance.
        • 'Fy' Yield force
        • 'k0' Initial stiffness of hysteretic component
        • 'dir' The direction of action of this link
      • name=HDRBYamamoto2DLink, HDRBYamamoto3DLink:
        • 'De' External rubber diameter.
        • 'Di' Internal rubber diameter.
        • 'Hr' Rubber bearing height.
      • name=lin2DTruss2, lin2DTruss3, lin3DTruss2, lin3DTruss3, kin3DTruss2:
        • 'material' is the hight of the line cross section.
        • 'A' is the cross section area.
      • name=lin2DQuad4, kin2DQuad4, lin2DQuad8:
        • 'material' is material identifier.
        • 'th' is the thickness of a quad element.
        • 'np' Number of quadrature-points used for the integration.
      • name=lin2DFrame2, lin3DFrame2:
        • 'section' is section identifier.
        • 'form' is the beam formulation (Bernoulli, Timoshenko).
      • name=null2DFrame2, null3DFrame2:
        • Nothing else needs to be specified
      • name=lin3DShell4:
        • 'section' is section identifier.
        • 'np' Number of quadrature-points used for the integration.
      • name=PML2DQuad4, PML2DQuad8, PML3DHexa8:
        • 'material' is material identifier.
        • 'n' is the polynomial degree.
        • 'th' is the thickness of a quad element.
        • 'L' is the Thickness of the PML region.
        • 'R' is the Reflection coefficient.
        • 'x0' is the PML coordinate of the reference point.
        • 'npml' is the vector of the outward normal.
        • 'np' Number of quadrature-points used for the integration.
      • name=lin3DHexa8, kin3DHexa8, lin3DHexa20:
        • 'material' is material identifier.
        • 'np' Number of quadrature-points used for the integration.

    Example

    A SECTION can be defined using the python interface as follows:
    SVL.addElement(tag=1, conn=[1,2], name='ZeroLength1D', attributes={'material': 1, 'dir': 1})
    SVL.addElement(tag=2, conn=[2,5], name='lin3DTruss2', attributes={'material': 1, 'area': 0.25})
    SVL.addElement(tag=3, conn=[1,5], name='lin3DFrame2', attributes={'section': 1, 'formulation': 'Bernoulli', 'rule': 'Gauss', 'np': 5})
    SVL.addElement(tag=4, conn=[13,14,18,17,16,15,19,20], name='lin3DHexa8', attributes={'material': 1, 'rule': 'Gauss', 'np': 8})
    SVL.addElement(tag=5, conn=[5,6,7,8], name='lin3DShell4', attributes={'section': 2, 'rule': 'Gauss', 'np': 9})

  • addSurface(tag, etag, conn):

    • tag : The identifier of this surface, i.e., tag > 0
    • etag :The identifier of this element, i.e., tag > 0
    • conn : Connectivity array of this element surface (must belong to an element)

    Example

    A SURFACE can be defined using the python interface as follows:
    SVL.addSurface(tag=5, etag=2, conn=[5,6,7,8])

  • addDamping(tag, name, attributes):

    • tag : The identifier of this damping model, i.e., tag > -1
    • name : The damping class name (Free, Rayleigh, Caughey)
    • attributes : Specific properties for the created damping, for example
      • name=Free:
      • name=Rayleigh:
        • 'list' : The element identifier that share this damping
        • 'ak' : The stiffness proportinal factor used in Rayleigh Damping
        • 'am' : The mass proportinal factor used in Rayleigh Damping
      • name=Caughey:
        • 'list' : The element identifier that share this damping
        • 'ai' : Parameters that multiplies mass/stiffness matrix for the Caughey damping.

    Example

    A DAMPING can be defined using the python interface as follows:
    SVL.addDamping(tag=1, name='Rayleigh', attributes={'am': 0.1244195110, 'ak': 0.0007878958, 'list': 'ALL'})

  • addFunction(tag, name, attributes):

    • tag : The identifier of this function, i.e., tag > -1
    • name : The Seismo-VLAB function class name: name=TimeSeries,Constant,PlaneWave,GeneralWave
    • attributes : Specific properties for the created function, for example
      • 'dir' : The direction of the acting function
      • 'mag' : The magnitude of the load
      • 'file' : The file where the load is going to be loaded. The file location is with respect to the main python script being executed

    Example

    A FUNCTION can be defined using the python interface as follows:
    SVL.addFunction(tag=1, name='Constant', attributes={'mag': 10.00, 'dir': [0.0, 0.0, -1.0]})
    SVL.addFunction(tag=2, name='TimeSerie', attributes={'file': 'ricker.in', 'dir': [25000.0, 0.0, 0.0]})

  • addLoad(tag, name, attributes):

    • tag : The identifier of this load, i.e., tag > -1
    • name : The Seismo-VLAB load class name: name=PointLoad, ElementLoad, SupportMotion
    • attributes : Specific properties for the created load, for example
      • 'fun' (int) The function identifier
      • 'list' (list) The identifier where this load acts upon
      • 'type' (str) The type of element load:
        • type=Constant,TimeSeries (PointLoad)
        • type=Surcace,Body,PlaneWave,GeneralWave (ElementLoad)

    Example

    A LOAD can be defined using the python interface as follows:
    SVL.addLoad(tag=1, name='PointLoad', attributes={'fun': 1, 'type': 'TimeSerie', 'list': [1]})
    SVL.addLoad(tag=2, name='ElementLoad', attributes={'fun': 1, 'type': 'Body', 'list': [1,2,3,4,5]})

  • addCombinationCase(tag, name, attributes):

    • tag : The identifier of this combination, i.e., tag > -1
    • name : The combination case name
    • attributes : Specific properties for the created combination, for example
      • 'folder' : Name of the folder to store solution
      • 'load' : Load identifiers to be combined
      • 'factor' : Combinational factors

    Example

    A LOAD COMBINATION can be defined using the python interface as follows:
    SVL.addCombinationCase(tag=1, name='SoilColumn', attributes={'load': [1, 2, 3, 4], 'factor': [1.0, 1.0, 1.0, 1.0]})

  • addRecorder(tag, attributes):

    • tag : The identifier of this recorder, i.e., tag > -1
    • attributes : Specific properties for the created recorder, for example
      • 'name' : The recorder type: 'Node', 'Element', 'Material', 'Section', 'ParaView'
      • 'file' : The output's file name
      • 'ndps' : Number of digit of precision for solution
      • 'nsamp' : Number of point to skip when sampling the response
      • 'resp' : The response associated to 'name' to be recorded
        • Node = disp, vel, accel, reaction
        • Section = strain, stress
        • Element = strain, stress, strainrate, internalforce
      • 'pos' : Coordinate where to measure stress/strain in section
      • 'list' : The identifier list to be recorded

    Example

    A RECORDER can be defined using the python interface as follows:
    SVL.addRecorder(tag=1, attributes={'name': 'PARAVIEW', 'file': 'Animation.out', 'ndps': 8})
    SVL.addRecorder(tag=2, attributes={'name': 'NODE', 'file': 'Displacement.out', 'ndps': 8, 'resp': 'disp', 'list': [25, 26, 32]})

  • addSolver(tag, attributes):

    • tag : The identifier of this solver, i.e., tag > -1
    • attributes : Specific properties for the created solver, for example
      • 'name' : The solver's name
      • 'update' : ON for nonlinear, OFF for linear
      • 'option' : Solver's option parameters, see in Definitions.py the SolverOption dictionary
      • 'tol' : Iteration tolerance for iterative solver

    Example

    A SOLVER can be defined using the python interface as follows:
    SVL.addSolver(tag=1, attributes={'name': 'Eigen', 'update': 'OFF'})
    SVL.addSolver(tag=2, attributes={'name': 'PETSC', 'option': 'KSPBCGS', 'tol': 1E-12})
    SVL.addSolver(tag=3, attributes={'name': 'MUMPS', 'option': 'SYM', 'update': 'OFF'})

  • addAlgorithm(tag, attributes):

    • tag : The identifier of this algorithm, i.e., tag > -1
    • attributes : Specific properties for the created algorithm, for example
      • 'name' : The algorithm's name
      • 'nstep' : The algorithm's number of increment steps
      • 'cnvgtol' : Tolerance to accept the solution has converged
      • 'cnvgtest' : The converge test to apply, these can be
        • 'UnbalanceForce'
        • 'IncrementalDisplacement'
        • 'RelativeUnbalanceForce'
        • 'RelativeIncrementalDisplacement'

    Example

    A ALGORITHM can be defined using the python interface as follows:
    SVL.addAlgorithm(tag=1, attributes={'name': 'Linear', 'nstep': 1})
    SVL.addAlgorithm(tag=2, attributes={'name': 'Newton', 'nstep': 50, 'cnvgtol': 1E-06, 'cnvgtest': 'IncrementalDisplacement'})

  • addIntegrator(tag, attributes):

    • tag : The identifier of this integrator, i.e., tag > -1
    • attributes : Specific properties for the created integrator, for example
      • 'name' : The integrator's name
      • 'dt' : The integrator's time step
      • 'ktol' : Tolerance to set a stiffness matrix value kij = 0.0
      • 'mtol' : Tolerance to set a mass matrix value mij = 0.0
      • 'ftol' : Tolerance to set a force vector value fj = 0.0

    Example

    A INTEGRATOR can be defined using the python interface as follows:
    SVL.addIntegrator(tag=1, attributes={'name': 'Newmark', 'dt': 0.00100})
    SVL.addIntegrator(tag=2, attributes={'name': 'Static'})

  • addAnalysis(tag, attributes):
    • tag : The identifier of this analysis, i.e., tag > -1
    • attributes : Specific properties for the created analysis, for example
      • 'name' : The analysis's name
      • 'nt' : The integrator's time step

Example

A ANALYSIS can be defined using the python interface as follows:
SVL.addAnalysis(tag=1, attributes={'name': 'Static', 'nt': 1})
SVL.addAnalysis(tag=2, attributes={'name': 'Dynamic', 'nt': 901})

  • addSimulation(tag, attributes):

    • tag : The identifier of this simulation, i.e., tag > -1
    • attributes : Specific properties for the created simulation, for example
      • 'analysis' : The analysis identifier to be used
      • 'algorithm' : The algorithm identifier to be used
      • 'integrator' : The integrator identifier to be used
      • 'solver' : The solver identifier to be used

    Example

    A SIMULATION can be defined using the python interface as follows:
    SVL.addSimulation(tag=1, combo=1, attributes={'analysis': 1, 'algorithm': 1, 'integrator': 1, 'solver': 1})
