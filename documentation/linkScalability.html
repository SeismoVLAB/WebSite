Scalability

Scalability can be defined as a measure of a parallel system’s capacity to increase speedup in proportion to the number of processors. Scalability in parallel numerically intensive computing can be categorized in two groups:

  • Strong Scalability (Amdahl 1967)
    Strong scalability correspond to the speedup obtained for a fixed problem size when the number of processor is increased. In 1967, Amdahl pointed out that the speedup is limited by the fraction of the serial part of the software that is not amenable to parallelization. Amdahl’s law can be formulated as follows:

    \[ \mathcal{S}_p = \frac{1}{f + \frac{1-f}{p}} \,, \]

    where \(f\) is the fraction of the serial code, \(p\) is the number of processors. But, in practice we measure it as:

    \[ \mathcal{S}_p = \frac{t_1}{t_p} \,, \]

    where \(t_1\) is the elapsed-time using one processor, and \(t_p\) is the wall-time using \(p\) processors.
  • Weak Scalability (Gustafson 1988)
    Weak scalability correspond to the speedup obtained when both the number of processors and the problem size are increased. In 1988, Gustafson pointed out that the speedup on the approximations that the parallel part scales linearly with the amount of resources, and that the serial part does not increase with respect to the size of the problem. Gustafson’s law can be formulated as follows:

    \[ \mathcal{S}_p = f + (1 - f)p \,, \]

    where \(f\) is the fraction of the serial code, \(p\) is the number of processors. But, in practice we measure it as:

    \[ \mathcal{S}_p = p\frac{t_1}{t_p} \,, \]

    where \(t_1\) is the elapsed-time using one processor, and \(t_p\) is the wall-time using \(p\) processors.

We have limited the number of processor to 128, since SVL is intended to be used in Desktop and Server Machines.
