SeismoVLab

This file is in charge of creating the input SeismoVLAB (*.svl) file required in the Run-Analysis. It first reads the METIS Graph output Files, then select the elements of the the domain partition, and finally writes the partition files in SeismoVLAB format.

The SeismoVLAB (*.svl) output file is performed at ./SVL/01-Pre_Process/Source/SeismoVLab.py. In this file, the following main functions are defined:

  • WriteMeshPartition: This function writes the SeismoVLAB (*.svl) file. It first reads the METIS Graph output files, then select the domain partition elements, and finally writes the partition files (name.$.svl) in SeismoVLAB format.
  • WriteParaviewFile: This function writes the Paraview mesh file. This file displays properties such as Partition, Sections and Materials.

There are several functions that are defined to handle writing of the model components in an ordered way:

  • WriteElementPartition: This function appends the Element in the finite element mesh to the SeismoVLAB mesh input files.
  • WriteNodePartition: This function appends the Initial conditions in the finite element mesh to the SeismoVLAB mesh input files.
  • WriteSectionsPartition: This function appends the Section that are used to construct structural Element to the SeismoVLAB mesh input files.
  • WriteMaterialPartition: This function appends the Material that are used to construct either solid/structural Element to the SeismoVLAB mesh input files.
  • WriteSimulationPartition: This function appends the Simulation specification to run the analysis for a particular load combination pattern.
Note
The later list are just a few function. The rest can be found at the SeismoVLab.py python script.
Attention
If a new Element/Material/Section or another feature is added, it MUST be written in this file, and its format MUST be consistent to the one provided in the Parser in the Run-Analysis. In other words, there are three steps in adding a new feature:
  • Modify the python Parser file to be able to read the input values of the new feature. This may require to define a new python dictionary to store this information.
  • Modify the python SeismoVLab so that the new feature is written in the output SeismoVLAB (*.svl) file required in the Run-Analysis.
  • Modify the C++ Parser file to be able to read the input values of the new feature with the format defined in SeismoVLab.
