Constraint

The Constraint keyword imposes linear kinematic relations between different degrees of freedom in the model. This object provides member functions to obtain the degree-of-freedom to be constrained (slave), and the degree-of-freedoms to be combined (masters). In the figure an example of a rigid diaphragm node k is shown in which two slave degree-of-freedom at node p and q denoted by \(U_\textrm{1}^{\textrm{p}}\) and \(U_\textrm{1}^{\textrm{q}}\) in the direction 1 are represented as a function of \(u_\textrm{1}\), \(u_\textrm{2}\), and \(u_\textrm{3}\) master degree-of-freedom.

A Constraint is created using the built-in parse-structure provided in the Parser,

CONSTRAINT cTag dTag n free, value

Variable Description
cTag A unique ( \(\leq -2\)) constraint number identifier.
dTag The associated total degree of freedom identifier (slave) of the constraint.
n Number of master degree of freedom that defines the constraint.
free Free-degree-of-freedom identifiers (masters) to be combined.
value Coefficient values for each degree of freedom.
Attention
The cTag number ( \(\leq -2\)) identifier must coincide with the constrained free-degree-of-freedom number.
The Constraint information is used in the Assembler to construct the transformation matrix \(\textbf{T} \in R^{nTotal \times nFree}\).
The number n represents the number of pair <free ( \(u_i\)), value ( \(\alpha_i\))> that are used in the constraint, \(\displaystyle{U_j = \sum_{i=1}^{n} \alpha_i \, u_i}\).
Note
It is worth mentioning that the upper-case letter represents the total degree of freedom number to which the constraint will be enforced, and it will be identified with a negative identifier. In addition, the lower-case letter represent the free-degree-of-freedom numbers that define the constraint.

Example

A constraint of the form: \(U_{12} = 1.0 \, u_{5}\) is constructed as:
CONSTRAINT -3 12 1 5 1.00

A constraint of the form: \(U_{24} = 1.0 \, u_{12} + 4.0 \, u_{14}\) is constructed as:
CONSTRAINT -5 24 2 12 1.00 14 4.00
