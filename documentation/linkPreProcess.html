Pre-Analysis

The Pre-Analysis consists on the following modules:

  • Parser
  • Metis
  • Numbering
  • PlaneWave
  • SeismoVLab

The Pre-Analysis consists on a set of Python files that transform a simulation file into Run-Analysis format. This basic task is orchestrated by two main scripts: PreProcess.py that essentially loads the input file and creates python dictionaries; and SeismoVLab.py that tarsnforms the python dictionaries into Run-Analysis input files. In addition, the Python parser provided in SVL handles the degree-of-freedom numbering in Numbering.py, the domain partition in Metis.py, soil spatial variablility in ...py, and domain reduction forces in PlaneWave.py. Therefore, the Pre-Analysis can be fully replaced by another program as long as the output files are written using the Run-Analysis format.

The Pre-Analysis input file format, Simulation.txt is defined by fields. Each field must start and ends with a *NAME and *END keyword that defines the scope of the instance being defined. Inside this scope, variables are defined with a -variable declaration followed by its corresponding instantiation, i.e, name, number, path. Each variable are separated by commas , which allows this format to be order insensitive. However, tags or identifiers for each entity need to be declared at the begining of each line within the field. For example,

  • A 2D Node with coordinates *(-75, 100)*, 5 degree-of-freedom, and identifier 7:

    *POINT
    7, -ndof 5, -pos -75.00 100.00
    *END

  • A Elastic2DPlaneStrain material with identifier 1, elastic modulus 500.0, Poisson's ratio 0.25, density 2000.0:

    *MATERIAL
    1, -name Elastic2DPlaneStrain, -E 500.0, -nu 0.25, -rho 2000.0
    *END

  • A domain partition for 8 processors using a Plain degree of freedom numbering:

    *PARTITION
    -metis /usr/bin/mpmetis, -np 8, -ordering Plain
    *END

Note that -metis specifies the path where the mpmetis is located. Also, additional nodes as well as material can be defined inside the POINT and MATERIAL scope. More details about the Pre-Analysis format can be found at Parser.
