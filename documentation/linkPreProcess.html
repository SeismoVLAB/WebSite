Pre-Analysis

The Pre-Analysis has the following folders:

  • Core
  • Method
  • Parser
  • Metis

The main Pre-Analysis components are shown in the figure below:

In general, the Pre-Analysis consists on a set of Python routines that allows to construct a finite element model. This interface is designed to be user friendly and to handle large models in different formats. The main task is orchestrated by two main scripts: Core/SeismoVLAB.py that essentially loads all required modules to generate a model; and Core/Definitions.py that contains the data structure (dictionaries) that stores the finite element information to be transformed into Run-Analysis input files.

The Python routines provided in SVL handles the degree-of-freedom numbering in Core/Numberer.py, the domain partition in Core/Partition.py, soil spatial variablility in Core/RandomField.py, and domain reduction forces in Core/PlaneWave.py. Also, the Pre-Analysis allows the user to parse formats from other softwares. For example, ETABS (.e2k), SAP (.s2k), and gmsh (.mesh) are currently available. Only basic features (point, material, element, restriction) in these files can be translated into SVL format. Similarly, several routines are provided in the Method/Attach.py that allows the user to create a model using a Python interface. Then, a user can take full advantage of all the features avaible in Python to manipulate, modify and create a finite element model.

Attention
The Pre-Analysis can be fully replaced by another program as long as the output files are written using the Run-Analysis format.
