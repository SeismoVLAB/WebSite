Parser

The python Parser takes a simplified format that is both case and order insensitive. In general, this format can be modified by the user as long as the output files are written in Run-Analysis format.

The input file is defined by fields, for which each field starts and ends with a *NAME and *END keyword. These keywords define the scope of the variables being defined. Inside each field, variables are defined with a -variable declaration followed by its corresponding instantiation, i.e, name, number, path. Each variable are separated by commas '','' which allows the format to be not only order insensitive, but also case insensitive. However, tags (identifiers) need to be declared at the begining of each line within the field.

MODEL

A MODEL can be defined as follows:

*MODEL
-name <str> , -ndim int , -mass <str>
*END

where the *MODEL and *END keyword define the scope of the model field, -name is the name that the generated partitioned files and paraview files will take, -ndim= 1,2,3 the dimension of the model, and -mass= Lumped, Consistent is the mass formulation for the mass matrix, this is used in dynamic analyses, and can be ommited in static analyses.

PARTITION

In order to exploit the parallel computing capabilities of Seismo-VLAB, a PARTITION can be defined for the MODEL as follows:

*PARTITION
-metis <str> , -np int , -ordering <str>
*END

where the *PARTITION and *END keyword define the scope of the partition field, -metis=./Metis/mpmetis or the global path to the mpmetis program, -np declares the number of desired partitions of the finite element model, and -ordering = Plain, CutHill-McKee specifies the ordering scheme applied to the degree of freedom numbering (see Numbering for more details).

POINT

A POINT can be defined as follows,

*POINT
Tag , -ndof int , -pos <double>
*END

where the *POINT and *END keyword define the scope of the point field, Tag is a unique (positive) identifier number associated to the point, -ndof is the number of degree of freedom associated to the point, and -pos is the coordinate (position) in space. Note that the length -pos vector must coincide with -ndim variable specified in MODEL.

MATERIAL

A MATERIAL can be defined as follows,

*MATERIAL
Tag , -name <str> , ...
*END

where the *MATERIAL and *END keyword define the scope of the material field, Tag is a unique (positive) identifier associated to the material, -name is the class name of the matrial, and depending on this name different attributes can be specified. For example,

  • -name=Viscous1DLinear:
    -eta double, is the linear viscous coefficient.

  • -name=Elastic1DLinear, Elastic2DPlaneStress, Elastic2DPlaneStrain, Elastic3DLinear:
    -E double, is the elastic modulus of elasticity.
    -nu double, is the Poisson’s ratio.
    -rho double, is the material density. Can be ommited for static analysis.

  • -name=PlasticPlaneStrainJ2, Plastic3DJ2:
    -K double, is the bulk’s modulus.
    -G double, is the elastic shear modulus.
    -rho double, is the material density.
    -H double, is the hardening modulus.
    -Sy double, is the yield stress.
    -beta double, is the Integration parameter or the Kinematic/Hardening ratio.

  • -name=PlasticPlaneStrainBA, Plastic3DBA:
    -K double, is the bulk’s modulus.
    -G double, is the elastic shear modulus.
    -rho double, is the material density.
    -H0 double, is the hardening modulus.
    -Su double, is the Undrained soil strength.
    -beta double, is the Integration parameter or the Kinematic/Hardening ratio.
    -h double, is the Exponential hardening modulus parameter.
    -m double, is the Exponential hardening modulus parameter.

SECTION

A SECTION can be defined as follows,

*SECTION
Tag, -name <str>, -type <str>, -prop <str>, ...
*END

where the *SECTION and *END keyword define the scope of the section field, Tag is a unique (positive) identifier associated to the section, -name is the class name of the section, -prop=Line, Area defines the section property, -type=Plain, Fiber specifies the model type of section and depending on this name different attributes can be specified. For example,

  • -type=Plain:
    -mat int, is the material identifier for the cross section.
    • -prop=Line:
      • -name=Circular:
        -r double, is the radius of the line cross section.

      • -name=Rectangular:
        -h double, is the hight of the line cross section.
        -b double, is the width of the line cross section.

      • -name=Angle,Channel, WideFlange, Tee, RectangularTube:
        -h double, is the hight of the line cross section.
        -b double, is the width of the line cross section.
        -tf double, is the flange thickness of the line cross section.
        -tw double, is the web thickness of the line cross section.

      • -name=CircularTube:
        -re double, is the thickness of the area section.
        -ri double, is the thickness of the area section.

    • -prop=Area:
      • -name=ThinArea:
        -t double, is the thickness of the area section.

  • -type=Fiber:
    this feature is under development.

RESTRAIN

A RESTRAIN can be defined as follows,

*RESTRAIN
Tag, -ndof int, -dof <bool>
*END

where the *RESTRAIN and *END keyword define the scope of the restrain field, Tag is a unique (positive) identifier associated to the point to apply restrictions, -ndof is the number of degree of freedom of the point, and -dof is a vector of 0 (free) or 1 (restrained) that specifies the condition of the degree of freedom. Note that -ndof must coincide with the length of -dof.

CONSTRAINT

A CONSTRAINT can be defined as follows,

*CONSTRAINT
Tag, -name <str>, -snode int, -sdof int, -mnode <int>, -mdof <int>, -factor <double>
*END

where the *CONSTRAINT and *END keyword define the scope of the constraint field, Tag is a unique (positive) identifier associated to the degree-of-freedom to apply constraint, -name=Equal,General specifies the type of constraint. -snode is (positive) identifier of the slave node, -sdof is (positive) identifier of the slave direction, -mnode is a vector of (positive) identifiers of master nodes, -mdof is a vector of (positive) identifiers of master directions, and -factor is a vector of coefficients to combine the master degree of freedom.

DIAPHRAGM

A DIAPHRAGM can be defined as follows,

*DIAPHRAGM
Tag, -axis <char>, -list <int>
*END

where the *DIAPHRAGM and *END keyword define the scope of the diaphragm field, Tag is a unique (positive) identifier associated to the diphragm and is usually a number different from the pre-defined Points (since a diaphragm is an extra point in space), -axis=X,Y, or Z is the axis in which the diaphragm will act on, and -list is a vector of all the nodes identifiers that belong to this diaphragm

DAMPING

A DAMPING can be defined as follows,

*DAMPING
Tag, -name <str>, ..., -list <int>
*END

where the *DAMPING and *END keyword define the scope of the damping field, Tag is a unique (positive) identifier associated to the damping, -name is the class name of the damping model, and depending on this name different attributes can be specified. For example,

  • -name=Free:

  • -name=Rayleigh:
    • -a0 double, parameters that multiplies mass matrix for the Rayleigh damping.
    • -a1 double, parameters that multiplies stiffness matrix for the Rayleigh damping.
    • -list <int> is a vector of all the elements identifiers that share this damping model.

  • -name=Caughey:
    • -n int, parameters that multiplies mass matrix for the Rayleigh damping.
    • -ai <double>, parameters that multiplies stiffness matrix for the Rayleigh damping.
    • -list <int> is a vector of all the elements identifiers that share this damping model.

ELEMENT

An ELEMENT can be defined as follows,

*ELEMENT
Tag, -name <str>, -conn <int>, ...
*END

where the *ELEMENT and *END keyword define the scope of the element field, Tag is a unique (positive) identifier associated to the element, -conn is a vector (connectivity array) of node identifiers, -name is the class name of the element. Depending on this name different attributes can be specified, for example

  • -name=ZeroLength1D:
    • -mag int, is material identifier.
    • -dir int, is the direction of action (X,Y,Z = 1, 2, 3).

  • -name=UnxBoucWen2DLink, UnxBoucWen3DLink:
    • -alpha double, ...
    • -mu double, ...
    • -eta double, ...
    • -beta double, ...
    • -gamma double, ...
    • -Tol double, ...
    • -fy double, ...
    • -k0 double, ...
    • -alpha1 double, ...
    • -alpha2 double, ...
    • -dir int, ...

  • -name=HDRBYamamoto2DLink, HDRBYamamoto3DLink:
    • -De double, External rubber diameter.
    • -Di double, Internal rubber diameter.
    • -Hr double, Rubber bearing height.

  • -name=lin2DTruss2, lin2DTruss3, lin3DTruss2, lin3DTruss3, kin3DTruss2:
    • -mat double, is the hight of the line cross section.
    • -A double, is the cross section area.

  • -name=lin2DQuad4, kin2DQuad4, lin2DQuad8:
    • -mat int, s material identifier.
    • -th double, is the thickness of a quad element.
    • -np int, Number of quadrature-points used for the integration.

  • -name=lin2DFrame2, lin3DFrame2:
    • -sec int, is section identifier.
    • -form <str>, is the beam formulation (Bernoulli, Timoshenko).

  • -name=null2DFrame2, null3DFrame2:
    • Nothing else needs to be specified
  • -name=lin3DShell4:
    • -sec int, is section identifier.
    • -np int, Number of quadrature-points used for the integration.

  • -name=PML2DQuad4, PML2DQuad8, PML3DHexa8:
    • -mat int, is material identifier.
    • -n double, is the polynomial degree.
    • -th double, is the thickness of a quad element.
    • -L double, is the Thickness of the PML region.
    • -Rdouble, is the Reflection coefficient.
    • -x0 <double>, is the PML coordinate of the reference point.
    • -npml <double>, is the vector of the outward normal.
    • -np int, Number of quadrature-points used for the integration.

  • -name=lin3DHexa8, kin3DHexa8, lin3DHexa20:
    • -mat int, is material identifier.
    • -np int, Number of quadrature-points used for the integration.

FUNCTION

A FUNCTION can be defined as follows,

*FUNCTION
Tag, -name <str>, -type <str>, ..., -list <int>
*END

where the *FUNCTION and *END keyword define the scope of the function field, Tag is a unique (positive) identifier associated to the function, -name=Constant, TimeSerie, PlaneWave, GeneralWave is the name of the function that will generate some load, -type=Static, Dynamic specifies the nature of the load that will be generated, and -list is a vector with the identifiers of the Points or Elements that share this load.

  • -name=Constant:
    • -mag double, is the magnitude or intensity of the load.
    • -dir <double>, is direction of the load.

  • -name=TimeSerie:
    • -path <str>, is file from where the time history data will be loaded.
    • -dir <double>, is direction of the load.

  • -name=PlaneWave:
    • -theta double, is the angle with respect to the vertical axis.
    • -phi double, is the horizontal angle with respect to the X-axis.
    • -Vs double, is the soil’s half-space shear wave velocity.
    • -nu double, is the soil’s half-space poisson’s ratio.
    • -x0 double, is a reference coordinate where the surface is located.

  • -name=GeneralWave:
    • -Path <str>, is file from where the time history data will be loaded. File’s name must contain the token $ to identify the node in which the signal acts on.

LOAD

A LOAD can be defined as follows,

*LOAD
Tag, -name <str>, -fun int, -type <str>, -list <int>
*END

where the *LOAD and *END keyword define the scope of the load field, Tag is a unique (positive) identifier associated to the load, -name=PointLoad, or ElementLoad is the name of the load that will be generated, -fun is identifier of the function associated to this load, -type=TimeSerie,Body, Traction, PlaneWave, GeneralWave is related to an specific aspect of this load, and -list is a vector with the identifiers of the Points or Elements that share this load.

COMBINATION

a COMBINATION can be defined as follows,

*COMBINATION
Tag, -name <str>, -load <int>, -factor <double>
*END

where the *COMBINATION and *END keyword define the scope of the combination field, Tag is a unique (positive) identifier associated to the combination, -name is the name of the combination and it is used to generate the folder where the results will be stored, -load is a vector of load identifiers to be applied in the model, -factor is a vector of coefficients that specifies the intensity of each load.

RECORDER

A RECORDER can be defined as follows,

*RECORDER
Tag, -name <str>, -file <str>, -ndps <int>, -nsamp <int>, -resp <str>, -list <int>
*END

where the *RECORDER and *END keyword define the scope of the recorder field, Tag is a unique (positive) identifier associated to the recorder, -name=Paraview, Node, Element, or Section is the name of the feature to be recorded, -file is the file name where the solution is going to be stored, -ndps is the number if digits of precision, -nsamp is the sampling number, -resp is the response to be recorded, and -list is a vector with the identifiers of the Points or Elements to be recorded.

SIMULATION

A SIMULATION can be defined as follows,

*SIMULATION
Tag , -analysis <str> ... , -algorithm <str> ... , -integrator <str> ... , -solver <str> ...
*END

where the *SIMULATION and *END keyword define the scope of the simulation field, Tag is a unique (positive) identifier associated to the COMBINATION to solve, -analysis is the name of the analysis class, -algorithm is the class name of the algorithm to use, -integrator is the class name of the integrator to use, -solver is the class name of the solver to use. For example

  • For the analysis section, these flags can be specified
    • -nstep <int>, is the number of time steps to solve.

  • For the algorithm section, these flags can be specified
    • -nstep <int>, is the number incremental step to use withing a time step.
    • -cnvgtol double, is the tolerance allowed to assume convergence.
    • -cnvgtest <str>, is the convergence test to use to check convergence.

  • For the integrator section, these flags can be specified
    • -dt double, is the time step increment.
    • -mtol double, is the value for which \(m_{ij} \sim 0\).
    • -ktol double, is the value for which \(k_{ij} \sim 0\).
    • -ftol double, is the value for which \(f_i \sim 0\).

  • For the solver section, these flags can be specified
    • -type <str>, is the type of linear system solver (direct,iterative).
    • -cholmode <str>, is the Cholesky decomposition storage (ON,OFF).
