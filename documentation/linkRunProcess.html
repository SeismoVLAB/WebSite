Seismo-VLAB  1.3
An Open-Source Finite Element Software for Meso-Scale Simulations
Run-Analysis

The Run-Analysis consists on the following modules:

  • Geometry Module
  • Solution Module
  • Input/Output Module

In order to provide a software design that actually promotes the maintainability and extendability of Seismo-VLAB, we have employed a fewer number of classes avoiding the overuse of subclasses definitions and inheritance that are actually not needed, without compromising extendability. We use the fully optimized STL to handle containers of data and operations over it. We employ the fully-optimize Eigen C++ Template Library to carry out linear algebra manipulations. This library offers a great interface that allows the user to create, operate and transform Eigen::VectorXd and Eigen::MatrixXd objects as it were in a Matlab environment, incentivizing the modification and creation of other classes in a very simple manner. In addition, we are aware that manipulation of data in a Pre-Analysis level can become complicated in a low-level language, thus we provide with a Python interfaces so that model's input file, spatial variability analysis as well as parameter identification can be generated externally in a much cleaner and straightforward manner.

In order to optimize all the previous requirements, we have divided the structure of Seismo-VLAB into two main process: the Pre-Analysis and the Run-Analysis shown schematically in the Figure bellow.

Bacause flexibility is needed to generate input files, domain (Mesh) partition, degree of freedom numbering, spatial variability of soil/structure properties, and parameter identification, we have use a high-level language such as Python to handle these tasks. Python offers the possibility to manipulate externally the Run-Analysis in a very straightforward manner, and thus abstracting the user from complicated manipulation in a low-level language. The popularity and user-friendly features that Python provides, makes it the perfect choice.

On the other hand, the Run-Analysis is the core of Seismo-VLAB and needs to be fast and efficient; therefore, we have use a low-level language such as C++ to handle these requirements. The Run-Analysis performs the finite element analysis–i.e., generates the element matrices, assemble them together, solve the linear system, and stores the solution according to the user's specifications.

The previous figure shows the main abstract classes employed to develop the Run-Analysis. In this diagram, the diamond arrow that gets to an abstract class represents the connecting abstract class that is contained. The point arrow that gets to an abstract class represents the connecting abstract class is used in some manner. The diagram also represents in blue boxes the main classes (core) of Seismo-VLAB. These are divided into the Mesh, Analysis, and Recorder classes. The Mesh class contains the finite element model, and as one can see, the FE components are represented by the group of green boxes. These are: Node, Material, Section, Element, Load, and Constraint. In general, a set of Node and Material forms an Element. The stiffness/mass/damping matrices of the element are computed using numerical integration i.e.,–QuadratureRule. The Analysis class contains the instructions or modifications to be performed in the finite element model, thus the group of yellow boxes represents the components of the Analysis class of Seismo-VLAB, i.e., the LinearSystem, Algorithm, and Integrator are employed to carry out an analysis.

However, the structure represented in Figure~fig:Seismo-VLAB_Program} is only well-suited for single core machines, but its extension to parallel machines is straight forward using OpenMPI. These process is achieved at the Pre-Analysis using Metis (http://glaros.dtc.umn.edu/gkhome/metis/metis/overview). Metis allows to partition the finite element mesh in almost equal number of elements–i.e., minimizing the load imbalance. Then, each partition is sent to a different processor using OpenMPI. Since in OpenMPI each process will act independently, all the information required to generate the mesh partition in the k-th processor must be provided.

The later figure provides a conceptual representation of the main classes required to be partitioned. We have decided to generate the Mesh partitions and the degree of freedom numbering off-line, using once again the Pre-Analysis implemented in Python. This allows Seismo-VLAB to be run almost fully parallel, avoing the user to make mistakes by partitioning the domain manually. The Mesh partition can be easily checked using Paraview (https://www.paraview.org/) after executing the Pre-Analysis.
