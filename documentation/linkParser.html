Parser

The Parser class defines an instance to translate an input's format file into Seismo-VLAB objects. The user manual explains extensively how the objects can be constructed using the built-in Parser class.

The procedure to parse objects such as Node, Material, Element, Load, and Constraint that defines a finite element model were described in their respective sections. In addition, the solution of a LoadCombo object is performed using the Analysis object that requires a Linear System, Integrator, and Algorithm which were also described in their respective sections.

Attention
The Parser class is defined internally in Seismo-VLAB. The Parser object does not check correctness of the input values. Thus, an inappropriate path to a files may crush the program.
The input file used in the Prser class must be consistent, this is the order in which class are defined must be sequential. For instance a proper order will be:
  • For the mesh file: Model, Material, Section, Node, Element, Damping, and Load.
  • For the simulation file: LoadCombo, Recorder, Analysis, Algorithm, Integrator, and Linear System
Interchange the order may cause (core-dump) error, since the generation of some objects requires allocation of other classes first. For instance, Element can't be generated if Material has not been defined first.
