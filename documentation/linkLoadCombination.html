LoadCombo

The LoadCombo keyword defines how a set of Load objects are going to be combined for an specific analysis. A load combination is used in the Assembler combining all prescribed Load as

\[ \mathbf{C}_{\textrm{loads}} = \sum_{\textrm{i}=1}^{\textrm{N}_\textrm{c}} \alpha_\textrm{i} \mathbf{L}_\textrm{i} \,, \nonumber \]

where \(\mathbf{L}_\textrm{i}\) are the Load to be combined, and \(\alpha_\textrm{i}\) the combination factors.

A LoadCombo can be created using the built-in parse-function provided in the Parser as follows,

LOADCOMBO Tag name nc lTag, factor

Variable Description
Tag Unique load combination object identifier.
name Name of the load combination.
nc Number of loads to be combined.
lTag The {Load} object identifier.
factor The combination factor for the {Load} object.
Attention
The name is used to generate a folder where the solutions are going to be stored.
If no LoadCombo is specified in the model, i.e., no loads are present in the mesh (or mesh partition), then the parameter nc = -1 and neither lTag nor factor must be provided.

Example

A partition without loads, thus without load combination, is prescribed as:
LOADCOMBO 1 Earthquake -1

An earthquake load combination of the form \(C = 1.0\, L_1 + 1.2\,L_2 - 0.9\,L_5\):
LOADCOMBO 1 Earthquake 3 1 1.00 2 1.20 5 -0.90
