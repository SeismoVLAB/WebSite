An Open-Source Finite Element Software for Meso-Scale Simulations
Definitions.py

The Entities dictionary stores all information regarding the finite element simulation. Some of the fields defines are:

  • Nodes : Dictionary that stores all node information, some of the fiels defined here are:
    • ndof : Number of degree of freedom, i.e., ndof > 0
    • coords : Coordinates of the node, i.e., (x,y) or (x,y,z)
    • freedof : The free (with restrain/constaint) degree of freedom numbering of this node
    • totaldof : The total (without restrain/constaint) degree of freedom numbering of this node
  • Masses : Dictionary that stores all mass information, some of the fiels defined here are:
    • ndof : Degree of freedom number where the mass will be added, i.e., ndof > 0
    • mass : Mass values to be added to specific degree of freedom
  • Constraints : Dictionary that stores all constraints applied to a certain degree of freedom, some of the fiels defined here are:
    • name : Seismo-VLAB material class name
    • attributes : Specific properties assigned to the constraint, for example:
      • 'stag' : The slave node(s) tags to be constrained
      • 'sdof' : The slave degree-of-freedom to be constrained
      • 'mtag' : The master node(s) tags to be employed
      • 'mdof' : The master degree-of-freedom to be combined
      • 'factor': The combinational factors to be combined
  • Supports : Dictionary that stores all support motions applied to a certain fixed degree of freedom, some of the fiels defined here are:
    • type : Constant or TimeSerie
    • value : Displacement value if type=static
    • file : Displacement time serie file if type=dynamic
    • dof : Degree of freedom to apply the support motion
  • Materials : Dictionary that stores all material information, some of the fiels defined here are:
    • name : Seismo-VLAB material class name
    • attributes : Specific properties assigned to the created material
  • Sections : Dictionary that stores all section information, some of the fiels defined here are:
    • name : Seismo-VLAB section class name
    • model : model of the section, i.e., model=plain,fiber
    • attributes : Specific properties assigned to the created section
  • Elements : Dictionary that stores all element information, some of the fiels defined here are:
    • name : Seismo-VLAB element class name
    • conn : Connectivity array of this element
    • attributes : Specific properties assigned to the created element
  • Surfaces : Dictionary that stores all element surfaces, some of the fiels defined here are:
    • etag : The identifier of this element's surface, i.e., tag > 0
    • conn : CConnectivity array of this element surface
  • Dampings : Dictionary that stores all damping models, some of the fiels defined here are:
    • name : Seismo-VLAB damping class name
    • attributes : Specific properties assigned to the created element
  • Solvers :
    • name : Seismo-VLAB solver class name
    • update : ON for nonlinear, OFF for linear
    • option : Solver's matrix structure, or iterative algorithm name
    • tol : Iteration tolerance for iterative solver
  • Algorithms :
    • name : Seismo-VLAB algorithm class name
    • nstep : The algorithm's number of increment steps
    • cnvgtol : Tolerance to accept the solution has converged
    • cnvgtest : The converge test to apply, these can be
      • 'UnbalanceForce'
      • 'IncrementalDisplacement'
      • 'RelativeUnbalanceForce'
      • 'RelativeIncrementalDisplacement'
  • Integrators :
    • name : Seismo-VLAB integrator class name
    • dt : The integrator's time step
    • ktol : Tolerance to set a stiffness matrix value kij = 0.0
    • mtol : Tolerance to set a mass matrix value mij = 0.0
    • ftol : Tolerance to set a force vector value fj = 0.0
  • Analyses :
    • name : Seismo-VLAB analysis class name
    • nt :The number of time step to be evolved
  • Simulations :
    • analysis : The analysis identifier to be used
    • algorithm : The algorithm identifier to be usedd
    • integrator : The integrator identifier to be used
    • solver : The solver identifier to be used

Similarly, the dictionary Options stores all user's inputs or configuration. Some of the fields defines are:

  • numbering : The degree of freedom numbering scheme, these can be 'plain', 'cuthill-McKee', 'maximum-degree'
  • metispath : The path where the Metis program is located. Default is '/usr/bin/mpmetis' (Linux)
  • massform : The mass matrix formulation. These can be 'consistent' or 'lumped'
  • nparts : The number of partition to divide the finite element mesh
  • dimension : The model's dimension, this is '=2' (2D) or '=3' (3D)

Some other dictionaries are defined. For example: the VTKcolors dictionary stores data to generate the VTK visualization, the SVLclasses stores the information of the material, section, and element classes available in SVL. The SolverOption dictionary has the available option of the solver, and ConvergeTest dictionary the available test that can be performed during the Newton-Raphson method.
