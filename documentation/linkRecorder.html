Recorder

The Recorder keyword defines a solution output file. Currently, there are two object solution that can be recorded: Node, Section and Element.

Node Responses

A NODE Recorder can be created using the built-in parse-function provided in the Parser as follows,

RECORDER NODE file nsample precision response nn nodes

Variable Description
file The file name where the results are going to be stored.
response Depend on the object, see the NOTE bellow.
nsample The number of sampling points to record the solution.
precision The precision number to store the results.
nn The number of nodes to be recorded.
node The list of Node identifiers.

Attention
If the response is recorded for a node, then it can be either disp, vel, accel, reaction.

Section Responses

An SECTION Recorder can be created using the built-in parse-function provided in the Parser as follows,

RECORDER SECTION file nsample precision response npos position ne elements

Variable Description
file The file name where the results are going to be stored.
response Depend on the object, see the NOTE bellow.
nsample The number of sampling points to record the solution.
precision The precision number to store the results.
npos The number of coordinates specified.
position Vector with the section coordinates.
ne The number of elements to be recorded.
element The list of Element identifiers.
Attention
A SECTION for frame elements requires two coordinates ( \(x_3\), \(x_2\)), while a SECTION for shell elements requires only one coordinate \(x_3\).
The SECTION response = strain, stress are allowed, and they are computed at each integration point.

Paraview Animation

An PARAVIEW Recorder can be created using the built-in parse-function provided in the Parser as follows,

RECORDER PARAVIEW file nsample precision nfeatures

Variable Description
file The file name where the results are going to be stored.
nsample The number of sampling points to record the solution.
precision The precision number to store the results.
nfeatures The number of features (for elements) to be written in paraview.
Note
Parallel results are provided in separated files. Thus, the solution within each processor is named as: Animation_PARTp.q.vtk, where p represents the partition number (starting from 0 to np), and q the solution number (starting from 0 to nTimeSteps/nsample).
Attention
A nsample = n means the paraview solution will be stored every n time steps. Paraview files are heavy and a default sampling rate of five is assuumed.
A nfeatures = n means that the number of connectivities with its length of all elements in the partition is n. This parameter is internally used in Paraview.
The PARAVIEW option currently saves two types of output:
  • POINT: Response at each Node of the finite element model. Thus, displacement, velocity and accelerations are displayed. The point quantities are represented in global coordinates.
  • CELL: Response at each Element of the finite element model. Thus, stresses and strains are displayed. Note that these quantities are computed at the integration points, for now SeismoVLAB interpolates these results to the Element center. Additionally, solid elements (truss, quadrilateral, hexahedra) represents the cell quantities in local coordinates, thus

    \[ \text{Strains : } \textrm{XX} = \epsilon_\textrm{11}, \textrm{YY} = \epsilon_\textrm{22}, \textrm{ZZ} = \epsilon_\textrm{33}, \textrm{XY} = \epsilon_\textrm{12}, \textrm{YZ} = \epsilon_\textrm{23}, \textrm{XZ} = \epsilon_\textrm{13}, \nonumber \\ \text{Stresses : } \textrm{XX} = \sigma_\textrm{11}, \textrm{YY} = \sigma_\textrm{22}, \textrm{ZZ} = \sigma_\textrm{33}, \textrm{XY} = \sigma_\textrm{12}, \textrm{YZ} = \sigma_\textrm{23}, \textrm{XZ} = \sigma_\textrm{13}. \nonumber \]

    However, for sructural elements (frames, and shells) stresses are generalized section forces, and thus for frames:

    \[ \text{Strains : }\textrm{XX} = \epsilon_{11}, \textrm{YY} = \gamma_2, \textrm{ZZ} = \gamma_3, \textrm{XY} = \varphi_1, \textrm{YZ} = \kappa_2, \textrm{XZ} = \kappa_3, \nonumber \\ \text{Stresses : }\textrm{XX} = \textrm{N}_1, \textrm{YY} = \textrm{V}_2, \textrm{ZZ} = \textrm{V}_3, \textrm{XY} = \textrm{T}, \textrm{YZ} = \textrm{M}_2, \textrm{XZ} = \textrm{M}_3. \nonumber \]

    and for shell elements:

    \[ \text{Strains : }\textrm{XX} = \epsilon_{11}, \textrm{YY} = \epsilon_{22}, \textrm{ZZ} = \epsilon_{12}, \textrm{XY} = \kappa_1, \textrm{YZ} = \kappa_2, \textrm{XZ} = \kappa_{12}, \nonumber \\ \text{Stresses : }\textrm{XX} = \textrm{N}_1, \textrm{YY} = \textrm{N}_2, \textrm{ZZ} = \textrm{N}_{12}, \textrm{XY} = \textrm{M}_1, \textrm{YZ} = \textrm{M}_2, \textrm{XZ} = \textrm{M}_{12}. \nonumber \]
